# Introduction
First off, thank you for considering contributing to ExamCPP Tasks. It's people like you that make ExamCPP Tasks such a helpful repository.

If you think that tests (task) is incorrect, consider to make an issue.
If you have your own task, make a pull request with your task, and make an issue.

ExamCPP Tasks is an open source project and we love to receive contributions from our community — you! The main way to contibute is too add more sample tasks.

# How to contribute (step‑by‑step)

### 1. Found a bug or an incorrect test?
**Open an issue first.**  
Describe the problem clearly (what is wrong, expected behaviour, actual behaviour). Do not send a pull request for bugs without an issue, unless it's a trivial typo.

### 2. Have an idea for a new task?
Open an issue describing your task idea – this avoids duplicate work. Wait for feedback, then proceed with the steps below.

### 3. Fork the repository
Click the `Fork` button on the top right of the [ExamCPP Tasks repo](https://github.com/MCtop4ik/ExamCPP_Tasks).

### 4. Clone your fork locally
```bash
git clone https://github.com/YourRepo/ExamCPP_Tasks.git
cd ExamCPP_Tasks
```

### 5. Create a new branch
Use a descriptive name:
```bash
git checkout -b add-new-task
```

### 6. Add your task
Consider to leave **main.cc** in folder **TaskN**.

### 7. Commit your changes
```bash
git add .
git commit -m "Add new task: genius task"
```

### 8. Push to your fork
```bash
git push origin add-new-task
```
### 9. Open a Pull Request (PR)
- Go to the original repository on GitHub.
- Click Compare & pull request.
- In the PR description, briefly explain what your task covers and why it’s useful.
- Link the related issue (the one you opened for the bug or task idea).
